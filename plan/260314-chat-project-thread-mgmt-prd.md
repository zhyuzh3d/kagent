# 需求文档：Chat 项目与会话管理系统 (Project & Thread Management)

## 1. 概述 (Introduction)
为了提升用户在 `kagent` 聊天页面的使用体验，支持多场景对话分类与历史追溯，本方案旨在引入“项目 (Project) - 会话 (Thread)”二级管理架构。用户可以创建多个项目来区分不同的任务领域，并在每个项目下开启多个独立的对话会话。

## 2. 核心概念 (Core Concepts)
- **Project (项目)**: 用户侧的长周期容器，用于沉淀特定主题的上下文、文件和记忆。
- **Thread (会话/线程)**: 项目内的具体对话记录。每个 Thread 都是一次独立的 ASR/LLM/TTS 交互流。

## 3. 功能需求 (Functional Requirements)

### 3.1 侧边栏与布局 (Sidebar & Layout)
- **侧边栏开关与标题**: 
  - 将顶部 Header 的 "MT0" 文字替换为一个组合：
    1. **边栏开关按钮**：内嵌 SVG 图标，点击打开/关闭左侧列表。
    2. **当前路径展示**：形如 `项目名称 / 会话名称`。
  - 侧边栏采用悬浮或推挤布局，移动端优先考虑悬浮。
- **项目列表**:
  - 展示所有项目名称。
  - 自动展开当前激活 Thread 所属的项目。
  - 支持高亮显示当前激活的项目和 Thread。

### 3.2 项目管理 (Project Management)
- **基础操作**: 
  - **重命名**: 点击项目名称旁的编辑图标或通过右键菜单、双击进行重命名。
  - **删除**: 删除项目及其下属的所有 Thread（需二次确认）。
- **排序管理**:
  - 支持通过拖拽 (Drag & Drop) 改变项目的上下排列顺序。
- **自动创建**:
  - 当用户进入页面且无任何项目时，系统自动创建 "Default Project"。

### 3.3 会话管理 (Thread Management)
- **多轮对话本质**: 
  - Thread 是一个长周期的对话线，支持多轮对话。
  - 切换 Thread 时自动加载其所有历史消息。
- **基础操作**:
  - **新建 Thread**: 在项目下方或项目标题栏提供“新建会话”按钮。
  - **重命名**: 对单个 Thread 进行重命名。
  - **删除**: 删除特定 Thread。
- **排序与移动**:
  - **内排序**: 在同一个项目下，支持拖拽 Thread 改变顺序。
  - **跨项目移动**: 支持将 Thread 从项目 A 拖拽并释放到项目 B。
- **自动激活**:
  - 当用户进入项目且无 Thread 时，自动创建 "Default Thread" 并激活。
  - 点击 Thread 立即切换对话内容，并加载对应历史记录。

### 3.4 后端接口 (Backend API)
- **Project API**:
  - `GET /api/projects`: 获取项目列表（含排序信息）。
  - `POST /api/projects`: 创建项目。
  - `PATCH /api/projects/:id`: 更新项目（标题或排序索引）。
  - `DELETE /api/projects/:id`: 删除项目。
- **Thread API**:
  - `GET /api/projects/:id/threads`: 获取项目下的 Thread 列表。
  - `POST /api/projects/:id/threads`: 在指定项目下创建 Thread。
  - `PATCH /api/threads/:id`: 更新 Thread（标题、排序索引或所属 Project）。
  - `DELETE /api/threads/:id`: 删除 Thread。

## 4. UI/UX 设计规范 (Design Specifications)
- **视觉风格**: 延续当前深色极简风格 (Glassmorphism)。
- **交互设计**:
  - 拖拽过程需有清晰的视觉反馈（阴影、占位符）。
  - 项目标题栏右侧保留 `+` (新建 Thread)、`...` (更多操作) 按钮。
  - 图标使用内敛的内嵌 SVG 数据。
- **响应式**: 侧边栏在小屏设备上默认关闭，提供蒙层点击收回。

## 5. 存储设计 (Storage Design)
- **SQLite 表结构更新**:
  - `projects` 表新增 `order_index (INTEGER)`。
  - `threads` 表新增 `order_index (INTEGER)`。
- **数据一致性**: 删除项目时，后端必须联级删除相关 `threads` 和 `messages`。
